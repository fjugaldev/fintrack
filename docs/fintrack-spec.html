Gestión de Cuentas

Efectivo, bancos, tarjetas y wallets

FeatureDescripciónFase
🏦Múltiples cuentasEfectivo, cuenta corriente, ahorro, tarjeta de crédito, inversión, wallet cryptoMVP
💵Multi-monedaCada cuenta tiene su propia moneda; cada transacción se registra en la moneda nativa de la cuenta. El patrimonio total se muestra convertido a la moneda base del usuario. En MVP el tipo de cambio se guarda manualmente en la transacción; en Fase 2 se automatiza vía Frankfurter API.MVP
📊Balance en tiempo realSaldo actualizado automáticamente con cada transacción registradaMVP
🎨Personalización de cuentasNombre, color, icono y tipo de cuenta definidos por el usuarioMVP
🔄Transferencias entre cuentasMover dinero entre cuentas propias sin doble conteo. Si las cuentas tienen distinta moneda (cross-currency), el usuario ingresa el monto en cada lado (origen y destino) y el sistema almacena el tipo de cambio implícito en el campo exchange_rate de la transacción.MVP
💳Tarjeta de crédito con cicloSoporte para fecha de corte, límite de crédito y deuda actualFase 2
📈Tipo de cambio automáticoFetch diario de tasas via Frankfurter API (gratuita, sin API key, fuente BCE ~32 monedas). Para monedas no cubiertas por BCE (ARS, COP, CLP) se complementa con ExchangeRate-API (tier gratuito 1500 req/mes). Las tasas se almacenan en tabla exchange_rates y el campo to_amount de transferencias cross-currency se auto-rellena con el tipo del día (editable por el usuario).Fase 2
🗄️Historial de saldosGráfica de evolución del balance de cada cuenta en el tiempoFase 2
📤Archivado de cuentasOcultar cuentas inactivas sin eliminar su historialFase 2

Progressive Web App — Móvil

Experiencia nativa en iOS y Android sin App Store · Powered by Serwist (@serwist/next)

Serwist (@serwist/next): Sucesor de next-pwa, recomendado oficialmente por Next.js. Basado en Google Workbox, maneja Service Workers, precaching del app shell y estrategias de caching en runtime. Para el modo offline con datos, Serwist cachea la app para que cargue sin conexión, y se complementa con IndexedDB (via lib idb) para almacenar transacciones creadas offline localmente. Al reconectar, un mecanismo de sync detecta la conexión y envía los registros pendientes a Supabase.
FeatureDescripciónFase
📱Instalable como app (PWA)Añadir a pantalla de inicio en iOS y Android sin App Store. Manifest y Service Worker generados via @serwist/nextMVP
📷Captura directa de cámaraIntegración nativa con cámara del móvil para fotos de ticketsMVP
⚡Acceso rápido: añadir gastoBotón flotante para registrar un gasto en menos de 5 segundosMVP
🗄️Precaching con SerwistApp shell, rutas principales y assets estáticos precacheados automáticamente por Serwist al instalar el Service WorkerMVP
🔌Modo offline con datosSerwist cachea la app para carga offline. Las transacciones creadas sin conexión se guardan en IndexedDB (lib idb) como cola de pendientesFase 2
🔄Sync automático al reconectarAl detectar conexión (navigator.onLine + evento online), se procesan los registros pendientes en IndexedDB y se envían a SupabaseFase 2
📊Runtime caching de APIEstrategias Serwist (stale-while-revalidate / network-first) para cachear respuestas de API y mostrar dashboard offline actualizadoFase 2
🔔Push notificationsAlertas de presupuesto y precios via PWA push notifications gestionadas desde el Service Worker de SerwistFase 3
